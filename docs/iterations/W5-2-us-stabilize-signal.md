# W5-2 美股止跌回稳信号新增

## 背景

Steven 提出：美股日内大跌后出现探底回升，需要主动推送「止跌回稳信号」，方便判断隔夜 A 股开盘承接。之前只有「异动预警」（涨跌幅超过阈值推），没有专门的止跌检测。

## 需求

- 检测逻辑：**日内跌幅≥1%** + **从低点反弹幅度≥30%** → 推送提醒
- 30% 阈值含义：`(close - low)/(high - low) ≥ 0.3` → 说明日内低点有买盘承接，下跌动能衰竭
- 同一交易日同一指数只推一次（去重）
- 保留原有异动预警，止跌是新增提醒，不替代

## 技术实现

### 数据源改造

1. 发现原有 `int_xxx` 接口只提供收盘静态数据，无高低点 → 切换为 `gb_$xxx` 接口（新浪美股指数完整接口）
2. 改造 `parse_us_index()` 解析器：自动探测字段数量，兼容两种格式
   - `int_` 格式（4字段）→ 兼容原有解析（仅现价+涨跌幅，不支持止跌）
   - `gb_$` 格式（≥8字段）→ 解析出 `open`/`high`/`low` → 支持止跌检测
3. config 中更新 code 为新格式：
   - `int_dji` → `gb_$dji`
   - `int_nasdaq` → `gb_$ixic`
   - `int_sp500` → `gb_$inx`

### 新增检测逻辑

在 `us_market.py` 循环里增加：
```python
stabilization_alerts = []
# 条件：跌幅≥1% + 高低点存在 + 反弹≥30%
if (info["pct"] <= -1.0 and "high" in info and "low" in info 
    and info["high"] > info["low"] and info["close"] > info["low"]):
    rebound_pct = (info["close"] - info["low"]) / (info["high"] - info["low"])
    if rebound_pct >= 0.3:
        key = f"stabilize_{name}_{self.today}"
        # 去重，只推一次
        if not self.state.has(key) or self.force:
            收集到 stabilization_alerts
```

### 输出格式

新增独立区块，清晰醒目：
```
✳️ 【止跌回稳信号】
✳️ 止跌回稳: 标普500
  跌幅 -1.01% | 低点 7431.26 | 现价 7457.69
  从低点反弹 39%，接近今日振幅中上部
```

### 状态机
- 有止跌信号 → 哪怕没有异动预警，也会推送整份报告，不会静默
- 去重：同一个交易日同一个指数仅推送一次
